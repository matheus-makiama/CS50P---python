# CS50P Lecture 1 Recall: Conditionals

## What I Learned

In Lecture 1, I learned about **conditionals**.

Conditionals are a way to change a program’s behavior depending on the input or situation.

Basically, the program can ask:

> “If this is true, what should I do?”

And depending on the answer, it can run different code.

## Programming Is a Language

I still do not understand programming deeply yet, but I think one important idea is that programming languages are ultimately tools we use to communicate with computers.

Different programming languages may have different rules and syntax, but the main purpose is similar:

> We are telling the computer what we want it to do.

So, before thinking about the code itself, I think it is important to understand clearly what I want the program to do.

## Understanding the Problem First

There are usually many ways to approach the same problem in programming.

Because of that, I think the most important thing is not only knowing how to write code, but also understanding the goal first.

If I do not understand what I want to make, I cannot judge which code is better.

So before writing code, I should ask myself:

- What problem am I trying to solve?
- What input does the program need?
- What output should the program give?
- What conditions should change the program’s behavior?

This connects to the idea of **要件定義**, which means defining the requirements before building something.

# CS50P Python Learning

This repository documents my Python learning journey through Harvard's **CS50P: Introduction to Programming with Python**.

I write the exercises myself, then use AI to improve readability, organize notes, create review questions, and help me check my understanding.

## Why CS50P?

I originally started learning Python through Angela Yu's 100 Days of Code, but later switched to CS50P because it fits my learning style better.

This repository is where I continue my Python study with a more structured system.

## My Learning Method: WDRSATR

I use a study method I call **WDRSATR**.

### W — Watch the video

I watch the lecture carefully and try to understand the main ideas before writing too much.

### D — Do the exercise

I write the exercises and practice code myself.

After that, I may use AI to improve the readability of the code, add clearer comments, and make the file easier to review later.

I also ask AI to assign me a small project based on the lecture so I can practice using what I learned.

### R — Recall

After studying, I write what I remember in my own words for about 5 to 10 minutes.

The goal is not to write perfectly.

The goal is to check what actually stayed in my brain without looking too much at the notes.

### S — Summarize with AI

I use AI to help summarize the lecture in a clean and easy-to-review format, usually using HTML + CSS or Markdown.

This helps me organize the messy raw knowledge into something I can review later.

### A — Answer questions

I answer review questions generated by AI to check if I really understood the important parts of the lecture.

If I get something wrong, I create a mistake correction file explaining:

- What I got wrong
- Why it was wrong
- What the correct answer is
- How to think about it next time

### T — Teach someone

I try to explain what I learned to someone else, or write it as if I were teaching a beginner.

If I can explain it simply, that means I understand it better.

### R — Review

I review my notes once in a while so I do not forget what I learned.

Programming is not just about watching lectures.

It is about touching the keyboard, getting confused, fixing mistakes, and slowly turning confusion into skill.

### Goal

My goal is to build a strong foundation in Python before moving deeper into AI, automation, and software development.

I want to understand programming properly, not just copy code.

This repository is my record of that process.

## Small Note About My English

Some recall notes and review-question answers may have imperfect English.

Since many of these files are written quickly during study sessions, I focus more on remembering, explaining, and checking my understanding than on writing perfect sentences.

So if the English is not perfect, please cut me some slack.
